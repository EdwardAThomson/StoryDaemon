# Dev Log

## 2026-07-09

Attacked the one failure the June validation runs kept showing: arc-pressure tracks rising tension targets but cannot de-escalate for a resolution, because the planner keeps choosing tense events no matter what number it is aimed at. The fix is the arc-phase planner mandate (Phase 3): the target tension curve is now also read as a story phase (rising, peak, falling, resolution), and the planner gets a firm event-level mandate per phase (escalate, confront, resolve) instead of only a numeric target. The mandate covers both the Stage 1 strategic path and plot-first's `plan_for_beat` (injected into the Stage 3 tactical prompt, suppressed when the beat carries its own `tension_target`). The bounded tension rewrite is now skipped when the target drop is large enough that different events, not different prose, are needed (`rewrite_futile`), and `arc_phase` is recorded in the per-tick coherence rubric. Alongside this, a new design doc (`docs/BLOCKS_CONTRACTS_LANDING_SKETCH.md`) reconciles the 2025 blocks/contracts architecture proposal with what has actually shipped, cutting the remaining work into slices: contracts riding on plot beats as the MVP, then precondition pressure, bounded contract repair, scene skeletons, and a measured per-sub-block generation experiment.

The sketch's first slices did not stay on paper long. The same day, beat generation itself was taught the curve: each beat in a generated batch is scheduled at its future tick, given an interpolated tension target and a phase directive in the generation prompt, and authored targets are reconciled to the schedule after parsing (clamp to a plus or minus 2 band, fill when missing). On top of that, contracts Slice 1 landed: `PlotBeat` now carries pre/postconditions authored at beat-generation time from the closed checker vocabulary, sanitized in both generation paths (unknown checks and phantom refs dropped, tension conditions reconciled against the beat's own target), evaluated during beat verification (a pass upgrades confidence, a failure routes to the existing keep-pending and horizon-revision paths), and measured in the coherence rubric. The stale `contracts.json` store and `ContractManager` are retired. A validation re-run of the June descent scenario confirmed the mandate works where it matters: the final scene dropped from 8/10 to 6/10 against a target of 4, with the planner demonstrably choosing aftermath events instead of confrontations (`docs/progress_report_20260709.md`).

**Decisions & notes:** The mandate is gated by `coherence.arc_phase_mandate` (default True; False restores exact pre-mandate behavior for A/B runs), but the rubric records `arc_phase` ungated so on/off runs stay comparable. The mandate's known gap at commit time (plot beat generation not receiving the phase) was closed the same day by the beat-generation bridge. Contracts stay behind `generation.use_contracts` (default False) until a plot-first smoke run exercises them live. On contracts, the sanitizer enforces consistency with the beat's tension rather than prohibiting contracts from authoring tension conditions. Two validation findings worth carrying forward: the rising phase ran hotter than the June control (possibly the escalate mandate amplifying the genre's hot bias), and the resolution mandate's close-open-loops clause did not bite (8 loops opened, 0 closed), which is direct evidence for the planned loop-aging pressure.

## 2026-07-01

Ran a full documentation audit against the current code and fixed the drift. The README's CLI Commands section was missing the shipped `novel metrics` and `novel titles` commands, so those got documented. `spec.md` had accumulated several inaccuracies: it referenced a `location.update` tool that doesn't exist (corrected to the real `location.generate`, in both the tools list and the example plan), and its Tech Stack table and API note still described the system as "Codex CLI (GPT-5)" with "API support can be added later" (replaced with the reality of the shipped multi-backend LLM support: Codex / api / claude-cli / gemini-cli, plus hosted-llm). Finally, `beats_and_loops.md` had stale source line-number citations that rot as code moves; those were swapped for more durable file+function references.

**Decisions & notes:** Docs-only sweep, no production code touched. Preferring file+function references over line numbers in docs is a deliberate anti-rot choice worth applying elsewhere.

## 2026-06-20

Scoped test-coverage pass over two pure, self-contained modules to push them to full coverage with no production-code changes. `utils/file_ops.py` went from 61% to 100% — covering `load_schema` (success, missing-file, invalid-JSON paths), `save_prompt_to_file` (content handling, name sanitization, custom subfolder/extension, IOError wrapping), `write_json`'s Unicode preservation (`ensure_ascii=False`), and the reachable error-wrapping branches in `open_file`/`write_file`/`read_json`. `memory/plot_outline.py` went from 75% to 100% via a new `tests/unit/test_plot_outline.py` that exercises every `PlotOutlineManager` method plus `PlotBeat`/`PlotOutline` `to_dict`/`from_dict` round-trips. The suite grew from 261 to 299 passing.

**Decisions & notes:** Tests-only by design — no production code was touched. This is a first targeted pass at the two simplest modules; the picks were deliberately "pure and self-contained" to make 100% coverage cheap and safe.
